Simple app demonstrating the caching of futures which are returned by the Play WS API.

We are simulating an external service which returns 503/ServiceUnavailable on the first call and 200/OK with a JSON String on the second call. If we would simply cache the future the cache would contain the exception which is thrown because of the 503/ServiceUnavailable. Therefore we only put a value in the cache if the operation is successful.

To try it out please follow plotd which will return a 500/Internal Server Error on the first GET and 200/OK on subsequent GETs. You can see in the logs that the external service is contacted twice.

You can find a more detailed explanation at GitHub.
